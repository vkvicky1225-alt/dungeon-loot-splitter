Logic Explanation

The Loot Array as Source of Truth

The loot array declared at the top of script.js is the single source of truth for this application. It holds plain object literals, each with a name, value, and quantity property. Nothing on screen is authoritative — the screen only reflects what the array contains. If the array changes, the display must update to match.

Why Quantity Was Added

Quantity was added to the data model because loot often comes in multiples — three health potions, five gold ingots. Storing a quantity on the object lets a single entry represent multiple units without duplicating rows. Total loot is calculated as value × quantity per item, summed across the array — so 5 potions worth 10 gold each correctly contribute 50 gold to the total.

Where updateUI() Performs All Work

updateUI() is the only function that calculates totals or updates the DOM. It runs a traditional for loop to accumulate value × quantity across every item, then a second loop to render loot rows into the CSS Grid. It then calculates the per-member share, enables or disables the Split button, and shows or hides the results region. No other function duplicates this logic.

State Controls Visibility and Actions

updateUI() uses if statements to decide what is visible. When the array is empty the loot table is hidden and the empty-state message appears. When party size is invalid, split results stay hidden and the Split button stays disabled. Visibility is controlled by adding or removing the .hidden CSS class — JavaScript never sets style properties directly, keeping all visual rules in the stylesheet.

Conditional Logic Prevents Invalid State

Before any item enters the array, addLoot() checks that the name is not empty, the value is a non-negative number, and quantity is at least 1. updateUI() checks that party size is a valid integer of 1 or greater before performing the split. If any check fails the operation is aborted and an error message appears in the interface. Invalid data never reaches the array and the interface never displays a stale or meaningless result.
